Aura XLSL capabilities

The Aura_Full_Project.xlsl repo ships a full reference implementation of the XLSL runtime, reactive dependency engine, and Paperweb rendering layer.

Typesafe runtime Data connectors Paperweb UI

XLSL Evaluation Core

Formula engine & dependency graph

Parses XLSL sheets, builds a dependency graph, and keeps cells up to date as inputs, external data, and user interactions change.

  • Incremental recomputation with precise dirty tracking.
  • Support for custom functions and external evaluators.
  • Serialization / hydration for server & edge runtimes.

Paperweb Rendering

Sheets as layouts & components

Declaratively map ranges, named cells, and tables into web components, dashboards, and flows while preserving formula semantics.

  • Bind cell values, ranges, and formulas directly into UI props.
  • Bi-directional sync between controls and sheet state.
  • Layout presets: boards, dashboards, tables, forms.

Connectors & APIs

Bring in live data & services

Wire Aura to databases, HTTP APIs, and event streams. Expose XLSL sheets as programmable surfaces within your stack.

  • Data functions (e.g. AURA.GET, AURA.POST, AURA.QUERY).
  • Webhooks, streaming updates, and event triggers.
  • Secure config and environment-aware bindings.

aura.config.xlsl
YAML-like 
paperweb:
  pages:
    dashboard:
      source: "dashboard!A1:E32"
      layout: "auto-dashboard"
      bindings:
        mrr: "B1"
        active_users: "B2"
        churn_rate: "B3"

runtime:
  mode: "reactive"
  evaluation: "incremental"
  connectors:
    - type: "http"
      name: "analytics"
      baseUrl: "https://api.example.com"
